# GRF - Grist Framework (Custom Widgets)

A powerful, modular framework for building complex, interactive applications directly within [Grist](https://getgrist.com). This toolkit provides advanced data visualization and management capabilities that go far beyond standard Grist widgets.

## 🚀 Key Features

### 1. Advanced Table Widget
*   **Full CRUD**: Add, Edit, and Delete records with ease.
*   **Editing Modes**: Switch between **Excel Style** (inline editing) and **Drawer Style** (form-based editing).
*   **Batch Save**: Highlighted "dirty" cells with a "Save All" bar for efficient bulk updates.
*   **Master-Detail (RefList)**: Expand rows to reveal nested tables, cards, or Tabulator grids for related records.
*   **Virtual Actions**: A customizable `_actions` column for quick row-level operations (View, Edit, Delete).
*   **Styling Overrides**: Column-level control to ignore Grist's native header/cell styles or conditional formatting.

### 2. Responsive Card System
*   **Dynamic Layouts**: Responsive grid systems with customizable spacing and padding.
*   **Drill-down Navigation**: In-place loading of widgets with a navigation history stack and automatic "Back" button.
*   **Action Groups**: Icon-based action buttons with dynamic tooltips showing real-time relationship counts (e.g., "Tasks (5)").
*   **Drag & Drop**: Functional reordering of cards with instant persistence to Grist.

### 3. Universal Renderers
*   **JSON UI Protocol**: Render complex UI components (pills, badges, progress bars, stats) from simple JSON strings generated by Grist formulas.
*   **Image Renderer**: Support for rendering photos and graphics from direct URLs or Grist Attachment fields.
*   **Theme Support**: Fully compatible with Grist's Light and Night themes.

### 4. Modular Architecture
*   **GristTableLens**: A centralized business logic layer that handles all data fetching and mutations.
*   **Headless Ready**: Logic is decoupled from the UI, allowing widgets to run both inside the Grist iframe and in "headless" mode via a REST proxy.
*   **Event-Driven**: Robust internal event bus (`grist-event-bus`) for seamless communication between components.

## 🛠️ Usage & Configuration

### The Configurator
All widgets are managed through a central hub: `configurator.html`. 
1.  Add the Configurator widget to your Grist document.
2.  Use it to create named configurations (mappings, styles, and actions).
3.  Each configuration has a unique `configId`.

### Linking Widgets
To use a specific widget (e.g., CardViewer):
1.  Add the `CardViewer.html` widget to your page.
2.  Open the settings gear (⚙️) inside the widget.
3.  Select the desired configuration from the dropdown.

### JSON UI Protocol Example
You can create a "Dynamic UI" field using a Grist formula:
```python
import json
return json.dumps([
  { "type": "pill", "label": "Urgente", "color": "#ef4444", "textColor": "white", "icon": "icon-alert" },
  { "type": "stat", "label": "Progresso", "value": "75%", "subtext": "+5% hoje" },
  { "type": "progress", "value": 75, "color": "#10b981" }
])
```
Configure the field as a "Dynamic UI (JSON)" in the widget settings to see it rendered visually.

## 📂 Project Structure

*   `libraries/`: Core framework code (EventBus, TableLens, Renderers).
*   `widgets/`: Entry points for specific widgets (CardViewer, TableWidget, DrawerViewer).
*   `UniversalViewer/`: A multi-purpose visualizer that detects configuration types automatically (supports Cards, Tables, BSC, and Indicators).
*   `IndicatorsWidget/`: Specialized KPI and trend visualization.
*   `static/`: The production-ready, synchronized mirror of the framework.

## 🤝 Contributing
This framework is built to be extensible. To add a new renderer, create a file in `libraries/grist-field-renderer/renderers/` and register it in `grist-field-renderer.js`.
